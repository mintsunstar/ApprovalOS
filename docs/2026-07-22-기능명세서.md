## 0. 문서 안내

### 0.1 목적

ApprovalOS가 **무엇을 하는지**와, **지금 코드에서 동작하는지**를 기능 단위로 정의한다.

### 0.2 현재 구현 여부 표기

| 표기 | 의미 | 사용 기준 |
|------|------|-----------|
| **✅ 구현** | 로컬 데모에서 해당 기능이 end-to-end로 동작 | 화면 + 데이터 반영 확인 가능 |
| **◐ 부분** | 핵심 흐름은 동작, 일부만 스텁·미연동 | 본 기능은 쓸 수 있으나 한계 있음 |
| **▽ 스텁** | UI·토스트·대체 동작만. 본래 의도 미충족 | 데모용 자리표시 |
| **✕ 미구현** | 화면에 없거나 동작하지 않음 | 다음 라운드 |

> 아래 **「현재」** 열·섹션은 2026-07-22 코드 기준이다. Supabase 실연동·실메일·실결제와는 별개다.

### 0.3 액터

| 액터 | 설명 |
|------|------|
| 게스트 | 비로그인 |
| 멤버 | WS 소속 로그인 유저 |
| WS 관리자 | `users.role = admin` |
| 승인자 | `users.role = approver` (또는 승인선에 지정) |
| 운영자 | `admin_users` (플랫폼 콘솔) |

### 0.4 제품 한 줄

디자인 시안에 대한 **투표 · 댓글 · AI 요약 · 다단계 승인**을 워크스페이스 단위로 진행하고, 플랫폼 운영자가 WS/유저/플랜을 통제한다.

### 0.5 구현 현황 한눈에

| 구분 | 개수 (F-01~19) |
|------|----------------|
| ✅ 구현 | 11 |
| ◐ 부분 | 6 |

| ✕ 미구현 (단독 ID) | 0 — 미구현 항목은 각 기능의 **미구현 하위**에 기술 |
